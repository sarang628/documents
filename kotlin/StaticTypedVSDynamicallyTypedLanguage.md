한 줄 요약
- 정적 타입 언어: 컴파일(실행 전)에 타입을 체크함 → 더 안전, 더 엄격
- 동적 타입 언어: 실행 중에 타입이 결정/체크됨 → 더 유연, 더 빠르게 작성

1️⃣ 정적 타입 언어 (Static Typed Language)
- 특징
  - 변수의 타입이 미리 정해짐
  - 컴파일 시점에 타입 오류를 잡아줌
  - 코드가 길어질 수 있지만, 대신 안정적
- 장점
  - 🛡️ 버그를 미리 잡음 (실행 전에)
  - 🧠 IDE 자동완성/리팩토링 강력
  - 🏗️ 대규모 프로젝트에 유리

- 단점
  - ✍️ 코드가 좀 장황
  - 🔄 빠른 실험/프로토타이핑엔 답답할 수 있음

2️⃣ 동적 타입 언어 (Dynamically Typed Language)
- 특징
  - 변수에 타입 선언이 없음
  - 실행할 때 타입이 정해짐
  - 훨씬 자유롭고 코드가 짧음
- 장점
  - 🚀 개발 속도 빠름
  - 🧪 실험·스크립트·자동화에 좋음
  - ✨ 코드가 간결
- 단점
  - 💥 런타임 에러 가능성
  - 🕵️ 타입 관련 버그가 실행 중에 터질 수 있음
  - 📦 프로젝트 커질수록 관리 난이도 상승
